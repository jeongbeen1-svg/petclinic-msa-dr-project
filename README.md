일단 S3 각자 가진 버킷으로 수정해서 사용하도록 하기 위해 ci/cd는 사용하지 않았고 lock 파일도 올리지 않았습니다.
수동으로 가져가셔서 init & apply 하시면 됩니다. <br><br>
```diff
[AWS]
-> 네트워크(vpc, subnet, igw, ngw) + EKS + Bastion Host + rds + lambda + s3가 생성됩니다.
-> 현재 람다 함수를 이용한 azure 백업까지 구현해놨습니다.
  (빠른 생성은 람다 없는 버전을 사용해주세요)
[Azure]
-> 네트워크(vpc, subnet, ngw) + AKS가 생성됩니다.
```
-> 오류 발생 시 먼저 말씀해주세요 수정해서 다시 올리겠습니다.<br>

[현재 발견된 오류]
* EKS entry 관련 오류 -> 무시해도 됨
* helm 관련 오류 -> 다시 apply하면 됨 (다운로드 지연으로 종종 발생)
```diff
* 람다 삭제 오류
  -> 람다가 있는 코드 사용 시 100% 발생
  -> 콘솔에서 람다를 먼저 삭제, tf state에서도 수동으로 삭제
  -> 이후 람다의 네트워크 인터페이스 삭제 확인 후 destroy 진행
  (콘솔에서 생성/삭제에도 비슷하게 소요됨)
```
* S3 삭제 오류
  -> 안에 파일이 있으면 삭제가 안됨
  -> 주석으로 강제 삭제 코드 넣어놨으니 사용하셔도 됩니다
